Como é feito?

fonte: Manual de Padrões para Iniciação do Pix - Banco central

Essa integração foi utilizada no App de Pagamentos apresentado na seção "Work".

Existem duas meneiras de integrar o QRCODE Pix em uma aplicação:


  • Estático
    Gerando uma sequência de caracteres correspondente ao payload do QR Code no padrão BR code, conforme exemplo com quebra de linhas:

    000201 Payload Format Indicator
    2658 Merchant Account Information
    0014br.gov.bcb.pix
    "Chave Pix"
    52040000 Merchant Category Code
    5303986 Transaction Currency
    5802BR Country Code
    5913"Nome"Merchant Name
    6008"Cidade"Merchant City
    6207 Additional Data Field Template
    0503***
    63041D3D CRC16



  • Dinâmico
    Semelhante ao anterior, mas com varias possibilidades novas. Sendo necessário uma integração com algum Provedor de Serviços de Pagamento (PSP) para validar os códigos dinamicos.

    Cada cobrança deve ter um txid único, sendo usado também para consultar o status de pagamento caso necessário.
